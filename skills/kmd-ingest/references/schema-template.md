# KB Schema

The KB root is wherever this file lives — the directory can be named
anything (`kb/`, `knowledge/`, or the whole vault). To place it somewhere
non-default, declare it in a `.kmd.json` at the workspace root:
`{"root": "knowledge"}`.

## Directory taxonomy (relative to the KB root)

| Directory | Contains | Page `type` |
|---|---|---|
| `sources/` | Immutable raw material: scraped pages, whitepapers, transcripts. Append-only — never edited, never synthesized content. | — (not pages) |
| `entities/` | People, organizations, products, tools — one page per distinct entity. | `entity` |
| `concepts/` | Domain knowledge, topic explainers, how-things-work. Living pages that improve over time. | `concept` |
| `projects/` | One page per project: status, goals, links to work. | `project` |
| `decisions/` | ADR-style records: context, decision, consequences. | `decision` |
| `reports/` | Promoted point-in-time outputs (scans, analyses). | `report` |

`INDEX.md` is generated by `recompile_index.py` (ships with the kmd-ingest
skill) — regenerate after every ingest; never hand-edit it.
`LOG.md` is append-only via `kb_log.py` — never hand-edited.
`.lint/` holds lint reports (operational records, not pages).

## Naming

**Filenames are titles** — Obsidian displays the filename everywhere (tabs,
file explorer, graph), so name pages in natural language:
`GPU memory math for LLMs.md`, not `gpu-memory-math-for-llms.md`.
Date-prefix decisions and reports: `2026-07-09 Vendor scan.md`. Avoid the
characters Obsidian forbids in filenames (`/ \ : # ^ [ ] |`). Filenames are
link targets — rename only with a full link sweep. Sources keep the name
they arrived with (append-only).

## Page frontmatter (required on every page)

```yaml
---
type: entity | concept | project | decision | report
created: YYYY-MM-DD
updated: YYYY-MM-DD             # bump on every edit
author: <name-or-agent-id>      # last substantive reviser
confidence: high | medium | low
sources: ["[[sources/...]]"]    # provenance rule below; named `sources`
                                # because Obsidian recognizes it natively
tags: []
---
```

The body starts on the line immediately after the closing `---` — no blank
line (Obsidian renders it as extra space) — beginning with the H1.

**`confidence`** is the page's epistemic marker — how much to trust its
claims at query time: `high` = multiple independent or authoritative
sources, uncontroversial; `medium` = a single or opinionated source;
`low` = thin or contested provenance. Set it at ingest, let lint recalibrate
it (the confidence-downgrade pass), and hedge answers drawn from anything
below `high`.

**Provenance rule:** every page declares where its content came from.
External-world claims point into `sources/`; internal artifacts point at
their origin — another page, a dated note, the project or conversation that
produced them. Reverse rule: every file in `sources/` must be referenced by
at least one page — an unreferenced source is un-ingested intake work,
surfaced by lint.

## Writing rules

- **Compression over expansion** — condense facts scattered across sources;
  never mirror greppable source content into pages.
- **New page vs. edit** — new page only for a distinct, linkable concept
  other pages will reference; otherwise improve the existing page.
- **No synthesis as evidence** — your own outputs are never filed into
  `sources/`.
- **Sweep on every ingest** — update cross-references and related pages, not
  just the target page.

## Operations

All KB writes follow the **kmd-ingest** skill. Health checks follow the
**kmd-lint** skill. Query is a convention: route through `INDEX.md` / search
first, read only selected pages, file durable answers back as pages.

# Python Code Style

These pages hold the Python conventions referenced by `AGENTS.md`. Read the
page that matches the work you are doing instead of treating `AGENTS.md` as a
long-form style manual.

## Pages

1. [Imports and typing](./imports.md): import paths, annotation rules, and type
   boundary constraints.
2. [Python conventions](./python.md): Python style, docstrings, async patterns,
   error handling, and framework-specific rules.
3. [Comments](./comments.md): comment, docstring, and field comment
   expectations.
4. [Module layout and exports](./modules.md): file naming, package layout,
   `__init__.py` usage, and export rules.
5. [Testing and pull requests](./testing-and-prs.md): test conventions, fixture
   placement, PR expectations, and commit message guidance.
6. [Tool design](./tool-design.md): first-party tool contracts, model-facing
   results, error boundaries, docs, examples, and tests.
